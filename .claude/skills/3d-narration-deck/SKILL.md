# 3D 나레이션 발표 제작 스킬

대본 → 챕터·문장 분해 → 3D 월드 구성 → ssjvoice 굽기 → 단일 HTML 임베드 → 검증.
이 리포의 `presentation3d.html` + `scripts/build-presentation3d.js`가 완성 레퍼런스다.
새 발표를 만들 때 이 빌더를 복제·수정하는 것이 맨바닥 재작성보다 빠르고 안전하다.

## 0. 재료 확인

- **대본**: 사용자가 준 낭독 대본. `[화면]`/`[스크롤]` 큐가 있으면 그대로 챕터·문장 경계로 쓴다.
- **ElevenLabs API 키**: 대화에서 받거나 `ELEVENLABS_API_KEY` 환경변수. **절대 파일·리포에 저장 금지.**
- **보이스**: `ssjvoice` = voice_id `dChkTgjs2tPbb8OYH4OX` (계정 조회로 재확인 가능).
  모델 `eleven_multilingual_v2`, voice_settings `{stability:0.5, similarity_boost:0.8}`,
  output `mp3_44100_128`.

## 1. 대본 → 구조 분해

- 챕터 3~10개. 챕터마다 캡션(=나레이션 문장) 배열. **문장 하나 = 자막 하나 = mp3 하나 = 카메라 앵커 하나.**
- 전역 문장 인덱스 순서로 `audio/<prefix>-<i>.mp3` 명명. 문장을 삭제/추가하면 **뒤 번호 전부 시프트**해야 한다.
- 확정되지 않은 수치·단정(전원 참여, 인원수 등)은 사용자에게 확인받거나 완곡하게.

## 2. 발음 규칙 (say 텍스트) — 자막 표기와 분리

자막(cap)은 원문 표기, TTS 입력(say)만 치환한다:

| 표기 | 발음 |
|---|---|
| NIA | 엔아이에이 |
| CDSA | 씨디에스에이 |
| PoC / PM / AX / AI / HTML | 피오씨 / 피엠 / 에이엑스 / 에이아이 / 에이치티엠엘 |
| 48명 등 "숫자+명" | 한글 수사로 (마흔여덟 명, 스물네 명) |
| 따옴표 “ ” " | 제거 |

새 약어가 나오면 같은 원칙으로 한글 독음을 추가한다.

## 3. 3D 엔진 문법 (디자인은 자유, 구조는 고정)

단일 HTML + three.js(r147~150 UMD, base64 또는 인라인 내장). 레퍼런스 엔진 구성:

- **스크롤 축**: 챕터당 `SEC_VH=2.4` 화면 높이. 챕터 내부 `HOLD=0.68` 구간에서 문장 진행(stepP),
  나머지 32%는 다음 챕터로 카메라 이동(travel).
- **존(zone)**: 챕터별 3D 그룹을 공간의 서로 다른 위치에 배치(겹치면 z 깊이 -50대로 밀거나 y를 올림).
  오브젝트는 `userData.step`으로 문장 도달 시 스케일-인. **비인접 존은 `visible=false`**
  (마지막 조감 챕터만 전체 표시) — 배경 잔상이 조잡함의 주범.
- **카메라 2단계**: 챕터 앵커 `A[i]` + **문장 서브앵커 `SUB[i][j]`**. 서브앵커 보간은
  `u = ease(clamp((x-k-.55)/.45,0,1))` — 앵커에서 55% 머물다 이동. 이래야 자동 상영 정지
  지점이 앵커 정중앙에 앉는다.
- **줌 거리 공식**: fov 55°, 16:9 기준 카드가 화면 폭 ~80%를 채우려면 `distance ≈ card_w × 0.67`.
  "설명하는 대상은 화면을 채운다"가 원칙 — 멀리서 보이면 실패.
- **비주얼 킷**: 캔버스 텍스처 카드(makeCard)·텍스트(makeText)·와이어 구체(makeOrb)·
  튜브 화살표+패킷(makeArrow). 카드 텍스처는 1024px 폭, 좌측 악센트 바, 이모지 금지.
- **팔레트**: 모노톤(그레이 4단) + **골드(#e8c268) 단일 악센트**(핵심 강조에만). 바닥 그리드·
  먼지·글로우는 저채도·저투명도.
- **폰트**: Pretendard Variable woff2를 base64 `@font-face`로 내장하고, 3D 부트 전에
  `document.fonts.load("700 40px 'Pretendard Variable'", 샘플한글)`을 await. CDN dynamic-subset은
  캔버스 텍스트에 적용되지 않으므로 금지.
- **조작**: 스크롤 스크럽 / 드래그 오빗 / 클릭 설명 카드 / 도트 내비 — 레퍼런스 엔진 그대로.
- **팝-포워드 + 딤**: 현재 문장의 오브젝트는 카메라 방향으로 `min(1.0, 거리×0.12)`만큼 앞으로
  나오고 `renderOrder=999`, 같은 존의 이미 등장한 오브젝트는 opacity 32%로 딤. 카드가 겹쳐도
  설명 중인 카드가 항상 또렷하게 앞에 보이게 하는 필수 규칙(과한 팝은 카메라를 뚫으니 상한 1.0).

## 4. 나레이션 자동 상영 플레이어

- `AUDIO_DATA` 마커(`/*AUDIO_EMBED_START*/.../*AUDIO_EMBED_END*/`)에
  `scripts/embed-audio.mjs <html> <prefix->`로 mp3 base64 주입. 폴백: `audio/` 파일 → 브라우저 TTS.
- 문장 목표 스크롤: `targetY(i) = (chapter + HOLD×((j+0.62)/nCaps)) × SEC_VH×100vh`.
- **싱크**: 스크롤 트윈은 문장 시작 후 `min(dur×0.45, 3.6s)` 안에 도착시키고 나머지는 정지.
- **호흡**: 문장 사이 0.9초 간격, `audio.playbackRate=0.93`(+preservesPitch), 실측 duration으로
  타이밍 갱신(`loadedmetadata`).
- **스크럽 추적 재생**: 자동 상영 중 휠/터치는 정지가 아니라 "위치 이동"이다 — 현재 오디오만
  멈추고(engaged 플래그 유지), 스크롤이 650ms 잠잠해지면 그 위치에 가장 가까운 문장부터
  자동으로 이어서 재생한다. 완전 정지는 일시정지 버튼/Space로만.

## 5. 파이프라인 (스크립트)

```bash
# ① 나레이션 굽기 (say 텍스트로, 전역 인덱스 순)
ELEVENLABS_API_KEY=... node  # fetch 루프: scripts/bake-narration.mjs 참고

# ② 빌드 (템플릿 치환 방식 권장 — scripts/build-presentation3d.js 복제·수정)
node scripts/build-presentation3d.js

# ③ 오디오/폰트 임베드
node scripts/embed-audio.mjs presentation3d.html pt-
```

빌더는 순수 문자열 치환이므로 **치환 실패를 조용히 넘기지 말 것** — 앵커 미발견 시 throw.

## 6. 검증 체크리스트 (필수)

1. 모든 인라인 `<script>`를 `new Function()`으로 구문 검사.
2. Playwright headless(`--use-angle=swiftshader`)로:
   - 각 챕터 자동상영 정지 지점(`chapter + 0.68×(j+0.62)/n`)에서 스크린샷 → **대상이 중앙에 크게** 보이는지
   - `#npStart` 클릭 후 scrollY 증가 + 내장 오디오(`data:audio`) 재생 확인
3. 폰트 무결성: `base64,d09GMg`(wOF2 매직) 존재 확인.
4. 오디오 폴더 없는 위치에 단독 복사해 열어도 재생되는지.
5. swiftshader는 느려 카메라 lerp가 지연됨 — 스크린샷은 3초 이상 대기 후.

## 7. 산출물

- `<이름>.html` 단일 파일(폰트+오디오+라이브러리 내장, 10~15MB 수준) — 이것만 공유하면 됨.
- 리포에 커밋: html + `audio/*.mp3` + 빌더. main과 작업 브랜치 모두 푸시(사용자 요청 기준).
- 사용자에게 파일 전송 + 장면 스크린샷 몇 장.

## 피드백 루프 관례

사용자가 "챕터명 + 몇 번째 문장"으로 지적하면: 카메라는 SUB 앵커만, 발음은 해당 mp3만 재굽고
번호 시프트 여부를 확인한다. 문장 삭제 시: 캡션·SUB·COUNTS·mp3 번호를 함께 갱신.
